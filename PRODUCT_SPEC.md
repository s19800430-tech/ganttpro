## 1. 产品概述

GanttPro 是一款**单文件离线优先**的项目管理工具，核心价值主张：

| 维度 | 传统工具（Jira / MS Project） | GanttPro |
|------|-------------------------------|----------|
| 部署 | SaaS / 企业服务器 | 单 HTML 文件，可离线运行 |
| 上手时间 | 数天培训 | 5 分钟 |
| 协作 | 需要后端 | PubNub 零后端实时聊天 |
| 移动端 | 功能残缺 | 原生移动优先设计 |
| 费用 | 月租 $10–25/人 | 免费开源 |

### 适用场景

- 10–50 人的软件、建设、营销项目团队
- 需要甘特图但不想引入重型 ERP 的 PMO
- 外包团队与甲方之间的轻量进度共享
- 个人项目经理的随身工具（离线可用）

## 3. PMBOK 知识域映射

PMBOK 第七版定义 12 个项目管理原则和 8 个绩效域。GanttPro 覆盖如下：

### 3.1 项目进度管理（Schedule Management）— 核心域

GanttPro 最强的域。对应 PMBOK 7 「规划」绩效域。

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 定义活动 | 任务新增，支持类型：任务 / 里程碑 / 群组 |
| 排列活动顺序 | 依赖关系：FS / SS / FF / SF 四种 |
| 估算工期 | 工作日输入字段，自动跳过非工作日 |
| 制定进度计划 | 甘特图自动渲染，支持日/周/月三种视图 |
| 控制进度 | 基线对比（设置基线后灰色显示原计划） |
| 关键路径 | 一键高亮关键路径（红色标注） |

**工作日历设置：**
- 自定义工作日（周一至周日任意组合）
- 排班制（早班/中班/夜班）
- 自定义非工作日（节假日、停工日）

### 3.2 项目范围管理（Scope Management）

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 创建 WBS | 群组-任务两级（可多层）树形结构，支持缩进/提升 |
| WBS 编码 | 可开启 WBS 编号显示（如 1.2.3） |
| 范围确认 | 审批流程：PM 提交 → 相关方审批 |

### 3.3 项目资源管理（Resource Management）

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 识别资源 | 资源池管理（姓名、角色、成本/时） |
| 分配资源 | 任务负责人（支持多人），芯片式输入 |
| 资源平衡 | 工作负荷视图：热力图显示超负荷 |
| 团队建设 | 聊天频道、@提醒、在线状态 |

### 3.4 项目成本管理（Cost Management）

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 估算成本 | 任务级「计划成本」字段 |
| 控制成本 | 「实际成本」字段，自动计算偏差 |
| 项目预算 | 报告页汇总全项目成本 |

### 3.5 项目沟通管理（Communications Management）

PMBOK 强调：**沟通占项目经理工作量的 90%**。GanttPro 用 Slack 式频道解决这个问题。

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 规划沟通 | 内置 4 个频道覆盖常见沟通场景 |
| 管理沟通 | 实时消息、历史记录、@提醒 |
| 监控沟通 | 在线状态、任务变更自动广播 |
| 信息分发 | Markdown 报告生成、导出分享 |

### 3.6 项目整合管理（Integration Management）

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 制定项目章程 | 项目新建：名称、开始日期、描述 |
| 监控项目工作 | 报告仪表板：完成率、燃尽图 |
| 变更控制 | 操作历史（Ctrl+Z 撤销，最多 50 步） |

### 3.7 项目质量管理（Quality Management）

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 质量保证 | 任务附件/证明材料（Evidence 字段） |
| 质量控制 | 审批流程确认可交付成果 |

### 3.8 项目风险管理（Risk Management）

> 当前版本为轻量实现，路线图中规划专属风险登记册。

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 识别风险 | 任务批注（右键菜单 → 批注） |
| 监控风险 | 逾期任务自动标红，关键路径提示 |

### 3.9 项目干系人管理（Stakeholder Management）

| PMBOK 过程 | GanttPro 实现 |
|-----------|---------------|
| 识别干系人 | 资源池 + 聊天频道成员 |
| 干系人参与 | 分享只读链接（复制任务链接功能） |
| 监控参与 | 在线状态指示、@提醒 |

## 4. 功能详述

### 4.1 项目管理

#### 多项目切换

左上角项目下拉菜单支持：
- **新建项目**：名称 + 开始日期 + 描述
- **切换项目**：即时切换，数据独立存储
- **删除项目**：二次确认防误操作
- **颜色标签**：每个项目自动分配主题色

#### 项目视图设置

通过右上角「设置」菜单可配置：
- 工作日历（哪几天是工作日）
- 排班时间（影响工时计算）
- 自定义节假日
- 显示/隐藏基线
- 显示/隐藏甘特条日期标签
- 显示/隐藏工作负荷面板
- 主题配色（7 种 + 自定义 Logo 提取色）

---

### 4.2 任务与进度

#### 任务类型

| 类型 | 图标 | 说明 |
|------|------|------|
| 任务（Task） | 彩色条 | 基本工作项，有开始/结束/工期 |
| 群组（Group） | 粗黑条 | 聚合多个子任务，日期自动扩展 |
| 里程碑（Milestone） | ◆ 菱形 | 标记关键节点，只有日期无工期 |

#### 任务弹窗——黄金位设计

弹窗分「主区域（黄金位）」和「可折叠的高级选项」：

**主区域（始终可见）：**
- 任务名称（大号字体，权重高）
- 开始日期 / 结束日期 / 工作日数
- 负责人（多人，芯片输入，支持 @自动补全）
- 进度（数字输入 + 滑块，双向同步）

**高级选项（默认折叠，点击「更多选项」展开）：**
- 任务类型切换
- 父群组选择
- 依赖关系（FS / SS / FF / SF）
- 参考链接
- 证明材料/附件描述
- 审批状态
- 计划成本 / 实际成本

> **自动展开规则**：编辑已有高级字段的任务时，高级区域自动展开，避免用户找不到已填内容。

#### 依赖关系

支持 PMBOK 标准四种逻辑关系：

| 类型 | 含义 | 示例 |
|------|------|------|
| FS（完成→开始） | A 完成后 B 才能开始 | 施工完才能验收 |
| SS（开始→开始） | A 开始后 B 才能开始 | 并行任务有先后顺序 |
| FF（完成→完成） | A 完成后 B 才能完成 | 文档随开发同步完成 |
| SF（开始→完成） | A 开始后 B 才能完成 | 守旧系统退役 |

依赖线在甘特图中以带箭头曲线渲染，关键路径依赖线高亮为红色。

#### 进度计算规则

- **任务**：手动设置 0–100%，或勾选「完成」自动设为 100%
- **群组**：加权平均子任务进度（按工期天数加权）
- **里程碑**：完成/未完成二态

#### 任务操作

| 操作 | 入口 |
|------|------|
| 新增任务 | 工具栏「+ 任务」按钮 / 快捷键 `N` |
| 编辑任务 | 双击任务行 / 甘特条 |
| 拖拉移动 | 拖动甘特条横向移动日期 |
| 拖拉调整工期 | 拖动甘特条右侧手柄 |
| 缩进/提升 | 工具栏缩进按钮 / `Tab` / `Shift+Tab` |
| 右键菜单 | 右键任意行：编辑、在下方添加、缩进、批注、复制链接、提交审批、删除 |
| 撤销 | `Ctrl+Z`（最多 50 步） |

### 4.3 甘特图可视化

#### 视图粒度

| 视图 | PPD（像素/天） | 适用场景 |
|------|--------------|--------|
| 日视图 | 36px | 短期冲刺，细节排期 |
| 周视图（默认） | 20px | 多周项目，常规管理 |
| 月视图 | 8px | 半年以上项目全局鸟瞰 |

快捷键 `1` / `2` / `3` 切换，或工具栏下拉选择。

#### 甘特条样式

- **普通任务**：圆角矩形，颜色可自定义（7 种预设）
- **已完成任务**：绿色
- **逾期任务**：红色边框
- **基线对比**：浅灰色细条显示原计划，当前条与基线的偏差一目了然
- **关键路径**：`Ctrl+Shift+C` 高亮关键路径（红色任务条 + 红色依赖线）
- **甘特条日期**：可在设置中开启，直接在条上显示开始/结束日期

#### 今日线

蓝色竖线标注今日位置，`T` 键或工具栏「今日」按钮快速滚动到今日。

#### 周末/非工作日

灰色底色标注非工作日区域（含自定义节假日），视觉上与工作日明显区分。

#### 里程碑时间线

甘特图顶部显示里程碑轨道，独立于任务行，跨越整个时间轴，不被任务遮挡。

#### 工作负荷视图

甘特图右侧可展开「工作负荷」面板：
- 按成员分行
- 每日工作小时数以色块显示（深色 = 超负荷）
- 与甘特图时间轴对齐

### 4.4 资源管理

#### 资源池

从右侧菜单进入「资源管理」：
- 成员姓名、角色（PM / 开发 / 设计 / 测试 / 其他）
- 时薪（用于成本估算）
- 自动同步到任务负责人自动补全列表

#### 工作负荷均衡

工作负荷视图直观显示：
- 每人每日工时汇总
- 超过日工时上限的日期以红色标注
- PM 可据此调整任务日期或重新分配

---

### 4.5 成本管理

每个任务支持：
- **计划成本**：预算金额
- **实际成本**：已发生金额

报告仪表板自动汇总：
- 项目总预算
- 已发生总成本
- 成本偏差（CV = 计划 - 实际）
- 每个群组的成本小计

### 4.6 风险与审批

#### 审批工作流

适用于需要正式确认可交付成果的场景（对应 PMBOK「确认范围」过程）：

1. 任务负责人在任务弹窗填写「证明材料」
2. 点击「提交审批」（或右键菜单）
3. 任务状态变为「审批中」，甘特条显示橙色图标
4. 审批人查看后可批准或拒绝

#### 批注

右键任务 → 「批注」：为任务添加文字批注，批注显示在任务行旁边。适合记录风险、决策背景、变更原因。

### 4.7 团队协作（Slack 模式）

#### 设计理念

PMBOK 指出沟通管理是项目成功的关键，但传统项目工具把沟通和进度管理割裂。GanttPro 将两者内置在同一工具中，参照 Slack 的频道模型实现轻量团队通讯。

#### 频道架构

| 频道 | 用途 | 相当于 Slack |
|------|------|-------------|
| # 综合 | 日常通知、公告、闲聊 | #general |
| # 甘特图 | 任务变更自动广播，进度讨论 | #dev-updates |
| # 进度汇报 | 成员主动汇报、PM 查阅 | #standup |
| # 随意聊 | 非正式交流 | #random |

#### 实时消息

- **即时推送**：PubNub 点对点推送，延迟 < 100ms
- **消息历史**：每次进入频道自动拉取最近 40 条消息
- **消息类型**：普通消息 / 系统消息（任务变更广播）/ @提醒

#### @提醒系统

输入 `@` 后弹出在线成员下拉列表：
- 方向键导航，`Tab`/`Enter` 补全
- 被 @到的成员收到浏览器通知（需授权）
- 通知内容包含发送者姓名

#### 在线状态

聊天面板顶部「在线栏」实时显示：
- 当前在线的所有成员（绿点 + 姓名）
- 每 2 分钟自动心跳保活
- 成员关闭标签页后 2 分钟内从在线列表消失

#### 任务广播

在 # 甘特图 频道自动广播：
- 任务创建：`📌 张三 创建了任务「UI 设计」`
- 任务更新：`📌 李四 更新了任务「接口开发」`

#### 通知角标

聊天 FAB 按钮（💬，右下角）显示红色未读数角标：
- 普通未读：显示数量
- @提醒：额外触发浏览器系统通知

#### 与 Slack 的对比与取舍

| 特性 | Slack | GanttPro | 取舍说明 |
|------|-------|----------|---------|
| 实时消息 | ✅ | ✅ | PubNub demo keys |
| 频道 | ✅ | ✅ 4个固定频道 | 精简，避免频道泛滥 |
| @提醒 | ✅ | ✅ | 完整实现 |
| 在线状态 | ✅ | ✅ | 心跳机制 |
| 消息历史 | 永久 | 最近 40 条 | demo key 限制 |
| 文件上传 | ✅ | ❌ | 路线图 |
| 线程回复 | ✅ | ❌ | 路线图 |
| Emoji 反应 | ✅ | ❌ | 路线图 |
| 搜索 | ✅ | ❌ | 路线图 |
| 与任务关联 | ❌ | ✅ 自动广播 | GanttPro 独有优势 |

## 5. 移动端体验

### 设计原则

> 移动端不是桌面端的缩小版，而是针对「随时查看、快速更新」场景的独立设计。

项目经理在会议室、施工现场、通勤路上使用手机的核心需求：
1. 快速查看某个任务的状态
2. 更新自己负责任务的进度
3. 看甘特图全貌
4. 和团队沟通

### 底部标签栏

| 标签 | 图标 | 功能 |
|------|------|------|
| 全部 | 📋 | 所有任务卡片列表 |
| 我的 | 👤 | 仅显示分配给当前用户的任务，含逾期/今日到期统计 |
| ＋ | （中央） | 快速新增任务 |
| 甘特图 | 📊 | 移动版甘特图（双列：固定标签 + 横向滚动时间轴） |
| 报告 | 📈 | 仪表板 |

### 任务卡片

每张卡片显示：
- 任务名（颜色圆点标识状态）
- 日期范围
- 负责人
- 进度百分比
- 进度条

逾期任务红色标签，今日到期黄色标签。

### 群组折叠

移动任务列表中，群组行可点击折叠/展开：
- `▼` 展开状态（默认）
- `▶` 折叠状态（CSS 旋转动画）

## 8. 数据与同步

### 本地存储

数据存储在浏览器 IndexedDB（通过 PouchDB），包含：
- 所有项目结构
- 所有任务及字段
- 用户设置（主题、工作日历等）
- 项目基线

同一设备同一浏览器数据自动持久化，刷新不丢失。

### 局域网同步（LAN Sync）

工具栏「LAN」按钮启动基于 WebSocket 的局域网广播：
- 同一 Wi-Fi 下其他成员打开同一 HTML 文件即可接收实时更新
- 无需服务器，适合团队会议室使用
- 状态指示：🟢 已连接 / 🟡 同步中 / 🔴 未连接

### PubNub 云同步（团队模式）

在 GitHub Pages 部署下自动启用：
- 聊天消息实时同步
- 未来版本计划：任务变更也走 PubNub 广播

### 数据安全

- 所有数据存储在用户本地浏览器，不上传至第三方服务器
- PubNub 仅用于聊天消息传输，不包含任务数据
- 支持 Excel 导出作为备份

## 9. 部署方式

### 方式一：本地直接打开

下载 `gantt.html`，双击在浏览器打开。完全离线，无需网络。

### 方式二：GitHub Pages（推荐团队使用）

访问：`https://geekx.github.io/ganttpro/`

优势：
- 团队所有人用同一 URL，自动进入「团队模式」
- PubNub 实时聊天自动启用
- 无需部署服务器

### 方式三：内网 Web 服务器

```bash
# 任意静态文件服务器
python3 -m http.server 8080
# 或
npx serve .
```

将 `gantt.html` 重命名为 `index.html` 放到服务目录即可。

### 方式四：Nginx / Apache

单文件静态托管，无需任何后端配置。

## 10. 已知限制与路线图

### 当前已知限制

| 限制 | 说明 | 影响 |
|------|------|------|
| PubNub demo keys | 公共 demo 频道，任何人可见 | 聊天无隐私保障，勿传敏感信息 |
| 聊天消息历史 | 最近 40 条（demo key 限制） | 长历史需升级到 PubNub 付费 key |
| 无用户认证 | 本地昵称自填，无法验证身份 | 适合信任团队内部使用 |
| 单文件大小 | ~470KB | 首次加载需稍等（缓存后瞬开） |
| 无文件附件 | Evidence 字段只能填文字/链接 | 需搭配云盘使用 |
| 循环任务月份 | 按 30 天近似 | 严格月份循环需日历计算 |
